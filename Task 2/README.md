# Задача 2. FFT/IFFT.

## Что реализовано:

- `FastFourierTransform` - класс быстрого прямого и обратного преобразования Фурье.
- Входные данные имеют тип `complex<double>`.
- Поддерживаются длины преобразования, которые раскладываются только на множители `2`, `3`, `5`.
- В `cpp/main.cpp` генерируются случайные комплексные входные данные.

Для каждой длины выполняется проверка:

```text
input -> forward FFT -> inverse FFT -> recovered
```

После этого программа сравнивает исходный вектор `input` и восстановленный вектор `recovered`.

## Запуск:

Проект рассчитан на запуск под Linux или WSL Ubuntu.

Нужны:

- `g++` с поддержкой C++17;
- `bash`.

Склонируйте репозиторий и перейдите в папку задания. В Linux или внутри WSL:

```bash
cd "<repo>/Task 2"
```

Если запуск идет из Windows PowerShell, можно перейти в папку Windows-путем:

```powershell
cd "E:\Users\Матвей\Desktop\Test.Yadro\Task 2"
```

Основной запуск:

```bash
bash run.sh
```

Из Windows PowerShell ту же команду можно выполнить через WSL:

```powershell
wsl -e bash run.sh
```

По умолчанию программа выбирает случайный seed, поэтому численные ошибки могут
немного отличаться при разных запусках. Для воспроизводимого результата можно
передать seed вручную:

```bash
bash run.sh 2026
```

Или из Windows PowerShell:

```powershell
wsl -e bash run.sh 2026
```

Скрипт проверяет наличие `g++` с поддержкой C++17, при необходимости
устанавливает его через `apt-get`, собирает программу, запускает ее и удаляет
временный исполняемый файл.

Если нужно собрать вручную без `run.sh`, используйте:

```bash
mkdir -p output
g++ -std=c++17 -O2 -Wall -Wextra -Iinclude \
    cpp/main.cpp cpp/FastFourierTransform.cpp \
    -o fft_task
./fft_task 2026
```

## Файлы результата:

После запуска создается папка `output`.

Основной файл для чтения:

```text
output/fft_report.txt
```

В нем сначала написано, что именно проверяется, а затем идет таблица с ошибками.

Дополнительный CSV-файл:

```text
output/fft_errors.csv
```

Он нужен, если результаты удобнее открыть в таблице или обработать отдельно.

## Что означают столбцы:

- `length` - длина преобразования `N`.
- `maximum_absolute_error` - максимальная ошибка по одному элементу:

```text
max |input[i] - recovered[i]|
```

- `root_mean_square_error` - среднеквадратичная ошибка:

```text
sqrt(sum |input[i] - recovered[i]|^2 / N)
```

- `relative_error` - относительная ошибка всего восстановленного вектора. Она показывает, насколько ошибка мала по сравнению с масштабом исходных данных:

```text
sqrt(sum |input[i] - recovered[i]|^2 / sum |input[i]|^2)
```

Чем ближе эти значения к нулю, тем точнее обратное преобразование восстановило исходные данные.

Для `double` нормальный результат в этой программе - ошибки примерно порядка `1e-15`.

## Формулы преобразования:

Прямое преобразование:

```text
X[k] = sum x[n] * exp(-j * 2*pi*k*n/N)
```

Обратное преобразование:

```text
x[n] = (1/N) * sum X[k] * exp(+j * 2*pi*k*n/N)
```

Деление на `N` выполняется один раз в методе `inverse`.
