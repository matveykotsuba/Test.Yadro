# Задача 3.

## Что реализовано:

- `ConvolutionalCode` - описание сверточного кода.
- `ConvolutionalEncoder` - кодер сверточного кода.
- `ViterbiDecoder` - hard-decision декодер Витерби.
- `BinarySymmetricChannel` - двоичный симметричный канал.
- `cpp/main.cpp` - моделирование BER для разных вероятностей ошибки канала.
- `python/plot_ber.py` - построение графика.

## Запуск:

Проект рассчитан на запуск под Linux или WSL Ubuntu.

Нужны:

- `g++` с поддержкой C++17;
- `python3`;
- `matplotlib`.

Если этих пакетов нет, `run.sh` попробует установить их через `apt-get`.

Склонируйте репозиторий и перейдите в папку задания. В Linux или внутри WSL:

```bash
cd "<repo>/Task 3"
```

Если запуск идет из Windows PowerShell, можно перейти в папку Windows-путем:

```powershell
cd "E:\Users\Матвей\Desktop\Test.Yadro\Task 3"
```

Основной запуск:

```bash
bash run.sh
```

Из Windows PowerShell ту же команду можно выполнить через WSL:

```powershell
wsl -e bash run.sh
```

По умолчанию программа выбирает случайные seed'ы, поэтому CSV получаются разные при разных запусках. Для воспроизводимого результата можно
передать два seed'а: первый для генерации информационных битов, второй для
ошибок в канале.

```bash
bash run.sh 12345 2026
```

Или из Windows PowerShell:

```powershell
wsl -e bash run.sh 12345 2026
```

Программа печатает использованные seed'ы в консоль.

Скрипт сам:

1. собирает C++ программу;
2. запускает моделирование;
3. удаляет исполняемый файл после запуска;
4. сохраняет CSV;
5. строит график.

Если нужно собрать вручную без `run.sh`, используйте:

```bash
mkdir -p output
g++ -std=c++17 -O2 -Wall -Wextra -Iinclude \
    cpp/main.cpp cpp/ConvolutionalCode.cpp cpp/BinarySymmetricChannel.cpp \
    -o viterbi_task
./viterbi_task 12345 2026
python3 python/plot_ber.py output/viterbi_ber.csv output/viterbi_ber.png
```

## Сверточный код:

Классы поддерживают код скорости `1/n`:

```cpp
ConvolutionalCodeConfig config{
    7,
    {0171, 0133}
};
```

Здесь:

- `7` - длина ограничения `K`;
- `{0171, 0133}` - порождающие полиномы в восьмеричной записи;
- количество полиномов задает число выходных битов на один входной бит.

В примере используется классический код rate `1/2`, `K = 7`.

## Модель канала:

Двоичный симметричный канал инвертирует каждый бит независимо:

```text
received_bit = transmitted_bit XOR Bernoulli(p)
```

где `p` - заданная вероятность ошибки в канале.

## Что делает программа:

1. Генерирует случайную последовательность информационных битов.
2. Кодирует ее сверточным кодером с добавлением `K - 1` хвостовых нулей.
3. Передает закодированные биты через BSC для разных значений `p`.
4. Декодирует принятые биты алгоритмом Витерби.
5. Считает BER на выходе декодера.
6. Записывает результат в CSV.
